# Методология разрабатываемого ПО

*Алгоритм многокадрового анализа серии изображений теневой съёмки для определения дисперсного состава и поля скоростей газо-капельного потока*

---

## 1. Общая концепция и принципиальное отличие от существующих подходов

Разрабатываемое программное обеспечение реализует многокадровый подход к обработке видеозаписей теневой съёмки газо-капельного потока. В отличие от систем типа ActualFlow, которые ограничены анализом отдельных пар кадров и тем самым подвержены статистическому смещению (Sampling Bias), предлагаемый метод обрабатывает непрерывные серии изображений — от нескольких сотен до нескольких тысяч кадров. Это позволяет строить репрезентативные статистические распределения по размерам капель и корректно восстанавливать поле скоростей, не теряя информации о траекториях быстрых частиц, успевающих покинуть поле зрения камеры быстрее, чем это можно зафиксировать попарным методом.

Обработка видеоряда включает два независимых, но взаимосвязанных аналитических модуля: **модуль дисперсионного анализа** (определение распределения капель по размерам) и **модуль велосиметрии** (построение поля скоростей методом многокадрового PTV). Оба модуля используют единый предобработанный стек кадров и формируют совместный отчёт по параметрам распыла.

---

## 2. Предобработка изображений

### 2.1. Вычитание фона

Первым и критически важным шагом является удаление статичного фона — неравномерной засветки, загрязнений оптики и артефактов осветителя. Фон вычисляется как покадровый минимум яркости по всей серии изображений (операция min-стекирования):

```
B(x, y) = min_t { I_t(x, y) }
```

Метод min-стекирования работает надёжно при условии, что ни одна капля не остаётся неподвижной в одной точке на протяжении всей записи — что справедливо для любого реального распыла. Вычитание фона выполняется по формуле:

```
I'_t(x, y) = I_t(x, y) − B(x, y)
```

Результатом является серия нормализованных кадров, содержащих только сигнал от капель на тёмном фоне.

### 2.2. Фильтрация шума и нормализация

После вычитания фона применяется медианный фильтр с ядром 3×3 пикселя для подавления импульсного шума матрицы камеры. Затем каждый кадр нормализуется по диапазону яркости, что обеспечивает инвариантность алгоритмов детектирования к изменениям интенсивности осветителя.

### 2.3. Бинаризация

Бинаризация выполняется методом адаптивного порога Оцу (Otsu thresholding) на каждом кадре независимо. Метод Оцу автоматически находит оптимальный порог разделения фона и объектов по критерию минимизации внутриклассовой дисперсии, что делает его устойчивым к неравномерной засветке и вариациям плотности распыла.

## 3. Модуль дисперсионного анализа

### 3.1. Детектирование капель

На бинаризованном изображении применяется алгоритм Кенни (Canny edge detection) для выделения контуров объектов. Параметры алгоритма (пороги гистерезиса) адаптируются автоматически на основе статистики каждого кадра. Замкнутые контуры заполняются морфологической операцией flood fill, после чего каждый связный объект (blob) регистрируется как кандидат на каплю.

### 3.2. Критерий резкости (фокусировки)

Поскольку теневая съёмка регистрирует также расфокусированные капли, находящиеся вне глубины резкости объектива, необходимо отделить резкие (in-focus) объекты от нерезких (out-of-focus). Для этого вычисляется градиентная метрика резкости в окрестности каждой обнаруженной капли:

```
S = σ(∇²G * I)
```

где ∇²G — лапласиан гауссова фильтра, σ — среднеквадратическое отклонение. Объекты с метрикой ниже адаптивного порога отбраковываются как расфокусированные и не включаются в дисперсионный анализ.

### 3.3. Измерение размеров

Для каждой прошедшей фильтр капли вычисляются следующие геометрические параметры:

- **Эквивалентный диаметр:** `d_eq = 2√(A/π)`, где A — площадь проекции капли в пикселях, пересчитанная в мкм через калибровочный коэффициент (мкм/пиксель);
- **Большая и малая полуоси** эллипса аппроксимации (метод моментов второго порядка);
- **Коэффициент формы (сферичности):** `φ = 4πA / P²`, где P — периметр контура (φ = 1 для идеальной сферы);
- **Координаты центроида** капли в системе кадра.

### 3.4. Многокадровое накопление и статистические распределения

Принципиальное преимущество предлагаемого метода заключается в накоплении данных по всей серии кадров. После обработки каждого кадра данные по размерам всех резких капель добавляются в общий массив. По итогам обработки всей серии вычисляются:

- Гистограмма числовой концентрации **N(d)** — распределение числа капель по размерам;
- Гистограмма объёмной концентрации **V(d)** — взвешенная по объёму `V ~ d³`;
- Характеристические диаметры: медианный объёмный **D_v0.5**, **D_v0.1**, **D_v0.9**;
- Диаметр Заутера `D₃₂ = Σd³ / Σd²`;
- Относительный разброс (relative span): `(D_v0.9 − D_v0.1) / D_v0.5`.

### 3.5. Поправочный коэффициент на скорость (коррекция Sampling Bias)

При двухкадровом анализе быстрые крупные капли, успевающие покинуть поле зрения между кадрами, систематически недопредставлены. В предлагаемом методе скорость каждой капли v_i определяется в модуле велосиметрии (см. раздел 4), после чего вводится поправочный коэффициент:

```
CF_i = v_i · DOF(d_i) · FOV_cor
```

где DOF(d_i) — глубина резкости для капли данного размера, FOV_cor — эффективная площадь поля зрения. Итоговый дисперсный состав вычисляется с учётом этих весовых коэффициентов, что устраняет статистическое смещение по скоростям и размерам.

## 4. Модуль велосиметрии: многокадровый PTV

### 4.1. Принцип метода

Поле скоростей восстанавливается методом Particle Tracking Velocimetry (PTV) в режиме многокадрового анализа. В отличие от классического двухкадрового PIV, который вычисляет осреднённое смещение ансамбля частиц в корреляционных окнах, многокадровый PTV отслеживает траектории отдельных капель через 4 последовательных кадра: **(i−1), (i), (i+1), (i+2)**. Это позволяет верифицировать парную идентификацию путём проверки кинематической непрерывности траектории, исключая ложные пары.

### 4.2. Алгоритм четырёхкадрового поиска

Для каждой капли, обнаруженной в кадре (i), выполняется следующая процедура:

1. В кадре (i+1) в радиусе r₁ (определяется из предварительной оценки максимальной скорости и временно́го шага Δt) формируется список кандидатов на совпадение;
2. Для каждого кандидата из кадра (i+1) экстраполируется ожидаемое положение частицы в кадре (i+2) и проверяется наличие там совпадающей капли в радиусе r₂ ≤ r₁;
3. Аналогично проверяется кадр (i−1): ожидаемое положение экстраполируется назад, и наличие капли там подтверждает валидность пары;
4. Пара принимается только при уникальном совпадении и наличии капли хотя бы в одном из проверочных кадров (i−1) или (i+2). При наличии подтверждения в обоих кадрах вектор скорости получает наивысший статус достоверности.

### 4.3. Критерии идентификации пар

Совпадение двух капель в соседних кадрах принимается при выполнении следующих условий:

- Геометрическое расстояние между центроидами не превышает r₁;
- Отношение диаметров обеих капель удовлетворяет условию: `0.8 < d_i / d_{i+1} < 1.25` (допуск на измерительный шум);
- Коэффициент формы обеих капель φ > 0.7 (фильтрация артефактов и нерезких объектов);
- Направление смещения согласуется с известным средним направлением потока (задаётся пользователем или определяется автоматически по первым 50 кадрам).

### 4.4. Вычисление векторов скоростей

После нахождения валидной пары скорость капли вычисляется как:

```
v = |r_{i+1} − r_i| / Δt · K
```

где Δt — временной интервал между кадрами (обратная величина частоты съёмки), K — калибровочный коэффициент масштаба (мкм/пиксель). Компоненты вектора скорости (v_x, v_y) сохраняются вместе с координатами точки измерения и диаметром капли.

### 4.5. Построение поля скоростей

Накопленные по всей серии векторы интерполируются на регулярную сетку методом взвешенного осреднения по ближайшим соседям (radius-based binning). В каждом узле сетки вычисляются среднее значение и стандартное отклонение скорости. Результат визуализируется в виде векторного поля с цветовой кодировкой модуля скорости. Области с менее чем N_min = 5 векторами помечаются как статистически ненадёжные.

## 5. Требования к исходной видеозаписи

| Параметр | Требование для дисперсионного анализа | Требование для поля скоростей |
|---|---|---|
| Частота съёмки | ≥ 500 кадр/с (заморозка движения) | ≥ 500 кадр/с; Δt должно соответствовать смещению 5–20 пикс. |
| Выдержка | ≤ 10 мкс (исключение смазывания) | ≤ 10 мкс |
| Пространственное разрешение | Минимальный диаметр капли > 3 пикселя | Минимальный диаметр трекируемой капли > 3 пикселя |
| Глубина резкости | Чётко ограничена; расфокусированные капли фильтруются | Достаточная для сохранения облика капли в кадрах (i) и (i+1) |
| Длина серии | ≥ 200 кадров (надёжная статистика) | ≥ 200 кадров |
| Плотность распыла | Перекрытие капель < 20% площади кадра | ≤ 10 капель на 100×100 пикс. (разреженный режим PTV) |
| Калибровка | Калибровочная метка в поле зрения или заранее | Та же калибровка масштаба |
| Формат файла | TIFF 8/16 бит, PNG, AVI без потерь | Те же требования |

## 6. Сводная схема обработки

1. Загрузка серии кадров → формирование стека изображений;
2. Вычисление фона (min-стекирование) → вычитание фона из каждого кадра;
3. Фильтрация шума (медианный фильтр) → нормализация яркости;
4. Бинаризация по методу Оцу → выделение объектов;
5. Детектирование контуров (Canny) → идентификация связных областей (blob labeling);
6. Применение критерия резкости → отбор in-focus капель;
7. Измерение геометрических параметров каждой капли (d, φ, x, y);
8. Многокадровый PTV: 4-кадровый поиск валидных пар → вычисление векторов скоростей;
9. Поправочный коэффициент CF на скорость (коррекция Sampling Bias);
10. Накопление данных по всей серии → статистические распределения N(d), V(d);
11. Интерполяция векторного поля на регулярную сетку → построение карты скоростей;
12. Формирование отчёта: D_v0.5, D₃₂, span, средняя скорость, векторное поле.

## 8. Заключение

Предлагаемая методология обеспечивает комплексный анализ теневых видеозаписей газо-капельного потока без применения дорогостоящих измерительных комплексов. Многокадровый PTV-подход с 4-кадровой верификацией траекторий принципиально устраняет основной недостаток попарного анализа — статистическое смещение вследствие потери быстрых частиц. Введение поправочного коэффициента на скорость при построении дисперсного состава делает результаты сопоставимыми с данными фазово-доплеровского анализатора (PDPA). Совместная реализация дисперсионного модуля и модуля велосиметрии в едином программном продукте, ориентированном на работу со стандартными видеокамерами, представляет собой конкурентное преимущество перед существующими отечественными и рядом зарубежных открытых решений.
